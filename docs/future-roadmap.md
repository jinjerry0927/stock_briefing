# Future Roadmap

Stock Briefing은 개인용 주식 관찰 도구이지만, 공부용 프로젝트로 계속 확장할 수 있다. 이 문서는 다음 업데이트 때 참고할 수 있는 아이디어 백로그다.

## 우선순위 높은 개선

### 1. 종목 상세 화면 고도화
- 현재가, 등락률, 뉴스, 차트를 더 구조적으로 배치한다.
- 52주 고가/저가, 최근 1개월/3개월 수익률, 거래량 변화율을 추가한다.
- 관심 메모와 보유 메모를 종목 상세에서 함께 보여준다.

학습 포인트: 데이터 모델링, API 조합, UI 정보 구조화

### 2. 차트 사용성 개선
- 마우스 hover 시 날짜, 시가, 고가, 저가, 종가, 거래량을 더 명확히 표시한다.
- 이동평균선 5일, 20일, 60일을 선택적으로 켜고 끌 수 있게 한다.
- 차트 범위를 1개월, 3개월, 6개월, 1년, 5년으로 확장한다.

학습 포인트: 시계열 데이터 처리, SVG/Canvas UI, 계산 지표

### 3. 포트폴리오 분석 강화
- 시장별, 통화별, 태그별 비중을 보여준다.
- 원화 환산 기준 총 평가금액과 달러 기준 평가금액을 함께 보여준다.
- 보유 종목 중 손익 기여도가 큰 종목을 자동으로 정리한다.

학습 포인트: 집계 로직, 데이터 시각화, 통화 환산

## 중간 난이도 기능

### 4. 거래 내역 기능
- 매수, 매도, 배당, 수수료를 기록한다.
- 평균단가를 거래 내역 기반으로 자동 계산한다.
- 실현손익과 미실현손익을 분리한다.

학습 포인트: 회계성 데이터 모델링, 상태 전이, SQLite 마이그레이션

### 5. 알림과 관심 가격
- 종목별 목표가, 관심가, 손절 기준가를 저장한다.
- 현재가가 기준에 가까워지면 대시보드에서 강조한다.
- 브라우저 알림 또는 로컬 알림은 후속으로 검토한다.

학습 포인트: 규칙 기반 알림, 사용자 설정, 브라우저 API

### 6. 데이터 캐시 관리 화면
- 시세/뉴스/환율 캐시 상태를 보여준다.
- 마지막 갱신 시간과 실패 원인을 확인한다.
- 수동 새로고침 버튼과 캐시 삭제 버튼을 제공한다.

학습 포인트: 운영 도구 UI, 캐시 정책, 오류 관찰성

## 학습 가치가 큰 확장

### 7. 공식 데이터 소스 교체 실험
- Alpha Vantage, FMP, OpenDART, Naver Search API 중 하나를 선택해 실제 연동한다.
- 무료 한도, 응답 품질, 지연 시간, 장애 대응을 비교한다.
- 현재 Yahoo/Naver 기반 로직을 provider interface로 분리한다.

학습 포인트: 외부 API 설계, adapter 패턴, rate limit 대응

### 8. 기술적 지표 추가
- 이동평균, 변동성, RSI, MACD를 계산한다.
- 지표는 매수/매도 추천이 아니라 관찰 정보로만 표시한다.
- 계산식과 데이터 한계를 문서화한다.

학습 포인트: 금융 데이터 계산, 순수 함수 테스트, 수치 검증

### 9. 백테스트 미니 모듈
- 특정 규칙을 과거 데이터에 적용해 결과를 확인한다.
- 예: 20일 이동평균 돌파, 52주 신고가, 단순 분할 매수.
- 수익률, 최대 낙폭, 승률을 계산한다.

학습 포인트: 시뮬레이션, 테스트 데이터, 성능 최적화

## 제품 완성도 개선

### 10. 온보딩과 샘플 데이터
- 처음 실행했을 때 빈 화면 대신 샘플 관심종목을 제안한다.
- 샘플 데이터와 실제 데이터 모드를 구분한다.
- 사용자가 직접 삭제하거나 시작할 수 있게 한다.

학습 포인트: 초기 사용자 경험, 로컬 상태 관리

### 11. 설정 화면
- 기본 시장, 기본 통화, 차트 기본 기간을 설정한다.
- 뉴스 표시 개수, 캐시 유지 시간, 화면 밀도를 설정한다.
- 설정값은 SQLite `settings` 테이블에 저장한다.

학습 포인트: 앱 설정 모델, 폼 UI, 지속성

### 12. 모바일/PWA 개선
- 모바일 폭에서 대시보드, 종목 상세, 포트폴리오를 탭 구조로 정리한다.
- PWA manifest와 아이콘을 추가한다.
- 오프라인 상태에서 캐시된 데이터를 볼 수 있게 한다.

학습 포인트: 반응형 UX, PWA, offline fallback

## 보안과 배포 확장

### 13. 개인 인증 모드
- 공개 배포를 고려할 경우 간단한 로그인 또는 접근 토큰을 추가한다.
- API route에 인증 middleware를 붙인다.
- 로컬 전용 모드와 배포 모드를 분리한다.

학습 포인트: 인증, 세션, 서버 보안

### 14. 배포용 DB 분리
- 로컬 SQLite와 배포용 데이터 저장소를 분리한다.
- 사용자별 포트폴리오 데이터를 분리 저장한다.
- 개인 데이터 백업/복원 기능을 설계한다.

학습 포인트: 데이터 격리, 배포 아키텍처, 백업 전략

### 15. 테스트 체계 구축
- API route의 정상/실패 케이스 테스트를 추가한다.
- 계산 함수는 단위 테스트로 검증한다.
- Playwright로 핵심 UI 흐름을 자동 확인한다.

학습 포인트: 테스트 전략, 회귀 방지, CI 준비

## 포트폴리오 정리 아이디어

- 문제: 종목 코드, 보유 현황, 뉴스, 차트를 한 화면에서 보기 불편했다.
- 해결: 개인용 로컬 주식 브리핑 대시보드로 관심종목, 보유종목, 환율, 차트를 통합했다.
- 기술 포인트: Next.js App Router, SQLite, API route, 캐시, 시계열 차트, 로컬 보안 정책.
- 다음 개선: 공식 데이터 소스 연동, 거래 내역, 기술적 지표, 테스트 자동화.

## 다음에 시작하기 좋은 순서

1. 설정 화면 추가
2. 이동평균선 추가
3. 포트폴리오 비중 차트 추가
4. 거래 내역 테이블 추가
5. 공식 API provider 하나 선택해 교체 실험
